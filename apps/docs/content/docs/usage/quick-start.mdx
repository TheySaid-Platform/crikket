This guide installs the Crikket Chrome extension and captures your first
bug report.

## Prerequisites

- A Crikket account and workspace
- Chrome (or a Chromium-based browser like Edge, Brave, or Arc)

<Steps>
  <Step>

  ### Install the Extension

  Install Crikket from the
  [Chrome Web Store](https://chromewebstore.google.com/detail/crikket/pjooelapomplepemjoebeanebffdieic).

  The extension landing card on your Crikket dashboard also surfaces the
  install link.

  If you self-host Crikket and prefer not to use the Web Store, see
  [Install Without Web Store](/docs/extension).

  </Step>

  <Step>

  ### Sign In

  Open the extension popup and sign in. The extension uses your existing
  Crikket session — no separate credentials needed.

  </Step>

  <Step>

  ### Record Your First Bug

  - Press `Alt+Shift+R` (or click the extension and pick **Record screen**)
    to start recording.
  - Reproduce the issue.
  - Press `Alt+Shift+S` to stop.

  The recording uploads to your workspace with console logs, network
  requests, and replay metadata attached.

  </Step>

  <Step>

  ### Verify the Report

  Open the report from your dashboard. Confirm:

  - the video plays
  - console / network panels show captured events
  - the share link works

  </Step>

</Steps>

## Notes

- For one-off screenshots, use `Alt+Shift+C` instead.
- Screen capture permission prompts are expected browser behavior on first
  use.
- If you self-host Crikket, point the extension at your own deployment via
  `VITE_APP_URL` / `VITE_SERVER_URL` (see
  [Install Without Web Store](/docs/extension)).
